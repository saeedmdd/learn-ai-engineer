Production LLM Checklist

A compact review sheet for every AI feature in this roadmap. If a feature cannot answer these questions, it is still a prototype.

1. Contract

  • User goal is stated in one sentence.
  • Allowed inputs and unsupported inputs are known.
  • Output schema is typed and validated.
  • Refusal behavior is explicit.
  • At least five examples cover normal and edge cases.

2. Model Call

  • Prompt separates instructions, user input, and external data.
  • Temperature and model choice are intentional.
  • Retries have limits and do not hide bad contracts.
  • Timeouts and rate-limit behavior are handled.
  • Token usage is logged or estimated.

3. Evals

  • Golden set has representative, edge, and adversarial examples.
  • Metrics match the product behavior, not just text similarity.
  • Prompt, model, and retrieval versions are recorded.
  • Regression thresholds are defined before changing the system.
  • Failures are inspected and grouped by cause.

4. Retrieval

  • Chunking strategy is documented.
  • Metadata filters are used where domain boundaries matter.
  • Answers cite source context.
  • Unsupported questions are refused.
  • Prompt-injection attempts inside documents are tested.

5. Agents

  • Every tool has a typed input and output.
  • Side effects are idempotent or guarded.
  • Risky actions pause for human review.
  • State can be inspected and resumed.
  • Tool failures are visible in traces.

6. Production

  • Secrets are not committed.
  • Auth, authorization, and tenant boundaries are clear.
  • Latency and cost budgets are defined.
  • Unsafe content and misuse cases are tested.
  • Monitoring shows quality, latency, cost, and errors.

Definition Of Done

An AI feature is done when it has a contract, a runnable eval set, observable traces, a known cost/latency profile, and documented behavior for unsupported or risky cases.
