AI Engineering Glossary

Use these terms consistently across lessons and projects. The definitions are intentionally practical: each term is framed by what an engineer must decide or verify.

LLM
A language model used as a software dependency. Treat it as nondeterministic infrastructure with latency, cost, quality, and safety constraints.
Token
The unit a model reads and writes. Tokens drive context length, latency, and cost, so token budgets are production constraints.
Prompt contract
The full agreement around a model call: task, inputs, context, output shape, refusal rules, and examples.
Structured output
A model response constrained to a known schema, such as JSON matching a typed object. Validation is part of the feature, not cleanup.
Tool calling
A pattern where the model chooses a typed function call and your application executes it. The application owns permissions and side effects.
Embedding
A vector representation of text or other data used for similarity search and retrieval.
Vector store
A database or index that retrieves embeddings by similarity, often with metadata filters.
RAG
Retrieval-augmented generation: retrieve relevant external context, then generate an answer grounded in that context.
Reranker
A second-stage model or scoring method that reorders retrieved documents for relevance before generation.
Eval
A repeatable check that measures whether an AI system meets expected behavior. Evals are regression tests for probabilistic systems.
Golden set
A curated set of representative examples with expected behavior. It should include normal, edge, and adversarial cases.
Hallucination
An unsupported or false model output. In production, define how the system detects, avoids, refuses, or contains it.
Prompt injection
An input that tries to override system instructions or misuse tools. Treat external content as data, not authority.
Agent
A stateful workflow where a model plans or chooses actions through tools. Agents need boundaries, traces, approval points, and recovery behavior.
Trace
A record of model calls, prompts, tool calls, retrieved context, outputs, latency, cost, and errors for one run.
Human in the loop
A deliberate pause for human review before a risky, irreversible, or high-impact action continues.
