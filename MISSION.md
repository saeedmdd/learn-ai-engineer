# Mission: Production AI Engineering

## Why
Become able to design, build, evaluate, and ship useful LLM-powered software as a software engineer. The goal is not to memorize AI terminology; it is to gain the judgment and hands-on patterns needed to put AI features into production responsibly.

## Success looks like
- Build AI features with typed inputs and outputs, tests, evals, tracing, and clear failure modes.
- Build a RAG assistant that answers from supplied sources and refuses unsupported claims.
- Build a tool-using agent with explicit state, permissions, human review, and recovery behavior.
- Ship a capstone app with a README, eval report, cost/latency budget, and deployment notes.

## Constraints
- Pace: about 6 hours per week.
- Learner profile: already experienced in software engineering.
- Bias: production LLM systems and AI applications before model training depth.
- API keys are not assumed; early projects should run locally without paid services where practical.

## Out of scope
- Deep ML research, architecture design, and training large models from scratch.
- MLOps for custom-trained models until the LLM application loop is solid.
- Broad survey learning that does not produce working systems or eval artifacts.
